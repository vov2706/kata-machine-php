# Kata Machine (PHP)

PHP-порт [kata-machine від ThePrimeagen](https://github.com/ThePrimeagen/kata-machine) —
тренажер для відпрацювання структур даних та алгоритмів.

Ідея: щодня ти отримуєш свіжу папку з **порожніми скелетонами** (`src/dayN/`) і
реалізуєш кожну кату з нуля. Спільний набір тестів перевіряє твій код. Повторення
формує впевненість.

## Вимоги

- **PHP 8.1+** (розроблялося на PHP 8.3). Перевірити: `php -v`.
- Composer не потрібен — PHPUnit лежить у репозиторії як `phpunit.phar`, а класи
  вантажить `bootstrap.php` (без `vendor/`).

## Структура

```
kata-machine-php/
├── kata.config.php        # конфіг: поточний `day` + список кат `dsa`
├── bootstrap.php          # автозавантаження Kata\* з папки поточного дня
├── phpunit.xml            # конфіг PHPUnit (вказує на src/__tests__)
├── phpunit.phar           # запускач тестів
├── scripts/
│   ├── dsa.php            # визначення інтерфейсів кат (джерело правди)
│   └── generate.php       # генерує папку наступного дня
└── src/
    ├── day1/              # ТВОЯ робота тут — по одному файлу на кату
    └── __tests__/         # набір тестів (не редагувати)
```

## Робочий цикл

### 1. Розв'язуй кати

Відкривай файли в `src/day1/` і заповнюй порожні тіла методів. Кожен файл — це
скелетон із правильною назвою класу, сигнатурами методів і підказками в docblock;
реалізацію пишеш ти. Приклад (`src/day1/BubbleSort.php`):

```php
class BubbleSort
{
    /**
     * @param int[] $arr
     */
    public static function bubble_sort(array &$arr): void
    {
        // твоя реалізація тут
    }
}
```

### 2. Запускай тести

Весь набір:

```bash
php phpunit.phar
```

Одну кату, поки над нею працюєш — за назвою (фільтром) або за шляхом до файлу:

```bash
php phpunit.phar --filter Queue
php phpunit.phar src/__tests__/QueueTest.php
```

Доки нічого не реалізовано, тести падають з `TypeError: ... none returned` (скелетон
нічого не повертає). Щойно ката правильна — її тест стає зеленим.

### 3. Починай новий день

Коли хочеш відпрацювати ті самі кати з чистого аркуша:

```bash
php scripts/generate.php
```

Ця команда:
1. створює наступну папку (`src/day2/`, далі `src/day3/`, …) зі свіжими порожніми скелетонами,
2. перепризначає `kata.config.php` (`'day' => 'dayN'`), щоб тести тепер ішли проти нового дня,
3. лишає всі попередні дні недоторканими як історію.

Далі повертаєшся до кроку 1 у новій папці.

> Генератор ніколи не перезаписує файл, який уже існує в папці цільового дня, —
> тож твоя незавершена робота в безпеці, навіть якщо запустиш його повторно.

## Налаштування набору кат

`kata.config.php` містить дві речі:

- **`day`** — з якої папки тести резолвлять класи `Kata\*`. `generate.php` оновлює це
  за тебе; зазвичай чіпати не треба.
- **`dsa`** — список кат для генерації. Додавай або прибирай записи, щоб змінити набір.

Кожна назва в `dsa` повинна мати відповідне визначення в `scripts/dsa.php` (цей файл
описує інтерфейс кожної кати — клас чи функція, сигнатури методів, типи повернення).
Щоб додати абсолютно нову кату: додай її інтерфейс у `scripts/dsa.php`, додай назву до
`dsa` в `kata.config.php`, потім запусти `php scripts/generate.php`.

## Як це працює разом

- `bootstrap.php` реєструє автозавантажувач, який мапить `Kata\Foo` →
  `src/<поточний-день>/Foo.php`, читаючи поточний день із `kata.config.php`. Це PHP-аналог
  аліасу `@code/*` з оригінального проєкту.
- Тести в `src/__tests__/*Test.php` завжди роблять `use Kata\<Kata>`, тож перемикання днів —
  це зміна одного рядка в конфігу (яку `generate.php` робить автоматично).
- `src/__tests__/Fixtures.php` містить спільні тестові дані (дерева, графи, вправа зі списком).
  Він не має суфікса `Test.php`, тому PHPUnit ніколи не збирає його як тест-набір.
